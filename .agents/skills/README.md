# 项目级技能包索引

本目录存放 Vibe Coding 规则包提供的**项目级技能（Full 档位可选）**。技能包用于补充 `AGENTS.md` 的执行方法，**不替代**强制底线，也不要求安装 OpenSpec、CodeGraph 或 Superpowers。

**Minimal 档位：** 无需复制本目录；仅 `AGENTS.md` 即可独立使用。

## 技能列表

| 技能 | 适用场景 | 触发方式 |
| --- | --- | --- |
| `executing-complex-requirements` | 复杂、高风险或跨模块产品需求，尤其是会改变行为、接口、数据、权限、状态流转、用户路径、验收标准、发布或回滚的变更 | 可由任务类型自动触发，也可由用户点名 |
| `updating-requirement-changes` | 已有需求的补充、范围变化、验收调整、需求正文更新；项目若启用 OpenSpec 则同步 change | 仅在用户明确点名或明确要求使用需求变化同步技能时触发 |

## 与可选工具的关系

| 工具 | 在本技能包中的角色 |
| --- | --- |
| OpenSpec | 已启用时，中等及以上需求优先用 `openspec/changes/` 承载；未启用时用 issue、PR、外部系统或轻量记录 |
| CodeGraph | 已安装时优先做影响分析；未安装时用 IDE 引用、语言服务器或人工阅读，并记录剩余风险 |
| Superpowers | 已安装时作为子流程（brainstorming、TDD 等）；未安装时按技能内检查清单与 AGENTS.md 默认推进 |

## 使用边界

- 本仓库是技能包和通用规则包，不是业务项目本身；不要求在本仓库初始化 `.codegraph/` 或 `openspec/`。
- `AGENTS.md` 是固定基线，不承载不同项目会变化的技术栈、业务规范或命令细节。
- 使用方项目的差异化规则应放在项目 README、`docs/规范/`、所选需求记录载体（OpenSpec / issue / 外部系统等）或项目自己的技能包中。
- 技能包创建或更新需求文档时，正文默认使用简体中文；代码标识、接口名、字段名、命令、外部专有名词和既有模板要求可保留英文。
- 根目录的 `SSO 单点登录复杂需求执行步骤示例.md` 是保留的教学示例，不是本仓库自身的 OpenSpec 需求记录。
