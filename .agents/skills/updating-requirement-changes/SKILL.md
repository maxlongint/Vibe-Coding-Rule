# 更新需求变化

## 概览

这个技能用于处理“已有需求”的补充、范围变化、验收调整和 OpenSpec change 同步。核心规则是：需求正文更新为当前有效口径，需求变化记录保留历史变化事实，避免直接覆盖历史。

## 触发边界

只在用户明确点名本技能，或明确要求“按需求变化同步技能处理”时使用。

不要在这些场景自动触发：

- 普通需求讨论。
- 规则问答。
- 临时想法或探索性设想。
- 尚未确认要更新已有需求记录的聊天内容。
- 新需求从零创建；这类任务应先按项目需求治理规则判断是否创建新的 OpenSpec change。

## 处理原则

- 先确认这是已有需求的变化，而不是全新需求、缺陷修复或普通讨论。
- 需求正文只保留当前有效口径，让后续实现者不用在旧口径和新口径之间猜测。
- 需求文档正文默认使用简体中文编写，包括 proposal、design、tasks、spec、变化记录和外部需求记录中的目标、边界、验收、风险和结论。代码标识、API 名称、字段名、命令和既有英文模板可保留英文。
- 变化记录必须追加，不要覆盖或删除历史变化。
- 记录变化时保留时间、来源、原口径、新口径、影响范围、验收变化和剩余风险。
- 如果变化会影响行为、接口、数据、权限、流程或验收标准，先同步需求记录，再改实现。
- 如果无法确认变化归属或有效口径，先向用户确认，不要猜测改写。

## 标准流程

1. 确认变化对象。
   - 找到已有 OpenSpec change、外部需求记录、任务记录或用户明确指定的需求文档。
   - 确认该需求当前状态是否允许继续变更。
   - 若原需求已完成、废弃或被替代，先判断是恢复原需求还是创建新需求。

2. 判断变化类型。
   - 补充说明：不改变目标，只补清约束、边界或解释。
   - 范围变化：新增、移除或替换能力范围。
   - 验收调整：改变通过条件、场景、反向断言或验证方式。
   - 方案调整：改变实现方案、数据流、权限规则、发布或回滚策略。
   - 记录同步：把用户确认、外部 issue 或评审结论同步回 OpenSpec。

3. 更新当前有效口径。
   - 更新 `proposal.md`、`design.md`、`tasks.md`、`spec.md` 或外部记录中的当前需求正文。
   - 当前正文应表达最新目标、边界、验收标准和剩余风险。
   - 当前正文默认使用简体中文；仅代码标识、接口名、字段名、命令、外部专有名词和模板要求保留英文。
   - 删除或改写过期口径时，确保变化记录已经保留原口径和变更原因。

4. 追加需求变化记录。
   - 在 OpenSpec change、任务记录、PR 描述、交付说明或项目指定位置追加记录。
   - 记录至少包含：

```text
时间：
来源：
变化类型：
原口径：
新口径：
影响范围：
验收变化：
实现影响：
验证调整：
剩余风险：
```

5. 同步任务和验收。
   - 更新任务状态、待办项和验证命令。
   - 增加或调整正向场景、反向场景和回归范围。
   - 如果变化影响模块、数据流、权限或状态流转，重新做影响分析。

6. 再进入实现。
   - 只有在当前有效口径、变化记录、任务和验收标准同步后，才修改代码。
   - 如果用户只是要求同步记录，不要顺手实现代码。

## OpenSpec 同步规则

使用 OpenSpec 时，优先按文件职责更新：

| 文件 | 更新内容 |
| --- | --- |
| `proposal.md` | 目标、非目标、范围、用户影响、风险、当前状态 |
| `design.md` | 方案取舍、数据流、权限、状态流转、发布和回滚 |
| `tasks.md` | 任务拆分、状态、验证命令、剩余风险 |
| `spec.md` | 外部可观察行为、验收场景、正向和反向断言 |

`spec.md` 不要写成内部实现清单；应描述用户、系统或外部接口可观察到的行为。

## 常见错误

| 错误 | 修正 |
| --- | --- |
| 直接覆盖需求正文，历史变化消失 | 当前正文更新为最新口径，同时追加变化记录 |
| 把临时想法当成需求变化 | 先确认来源和有效性 |
| 只改 proposal，不改 spec 和 tasks | 根据影响同步设计、任务和验收 |
| 验收标准变化后仍沿用旧测试 | 更新测试和验证命令 |
| 需求正文继续使用英文或中英混杂导致口径不清 | 默认改为简体中文，必要的代码、接口、字段和命令英文除外 |
| 原需求已完成后继续追加任务 | 先判断恢复原 change 还是创建新 change |
| 用户没有点名技能就自动套用 | 不自动触发，按普通需求治理规则处理 |

## 完成检查

- 已确认用户明确点名本技能或明确要求使用需求变化同步技能。
- 已确认变化属于已有需求。
- 当前有效口径已更新。
- 需求文档正文已使用简体中文，必要的代码标识、接口名、字段名和命令除外。
- 变化记录已追加，且包含时间、来源、原口径、新口径、影响范围、验收变化和剩余风险。
- OpenSpec 或外部记录的任务、设计和规格已按影响同步。
- 验证范围和剩余风险已说明。
