# 安全策略

## 支持的版本

本仓库为文档与规范包，无运行时服务。仅维护**当前 main 分支**上的最新版本。

## 报告漏洞

如果你发现本仓库存在安全问题（例如文档示例意外包含凭据模板、误导性安装步骤可能导致风险）：

1. **请勿**在公开 Issue 中披露细节。
2. 通过 GitHub [Security Advisories](https://github.com/maxlongint/Vibe-Coding-Rule/security/advisories/new) 私下报告，或向仓库维护者私信说明。
3. 提供：问题描述、受影响文件路径、复现或影响说明、建议修复（如有）。

维护者会在合理时间内确认并回复处理计划。

## 范围说明

本仓库不包含业务代码与生产凭据。报告内容应与**本规范仓库本身**相关；业务项目的安全问题请向对应项目维护者报告。
