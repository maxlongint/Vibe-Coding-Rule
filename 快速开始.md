# 快速开始

> 核心规范是 [`AGENTS.md`](./AGENTS.md)。按团队成熟度选场景；接入时建议一次性建好下列目录。

## 业务项目目录总览

```text
your-project/
├── AGENTS.md                 # 必做：从本仓库复制
├── docs/                     # 推荐：长期项目规范
│   └── README.md             # 模板：本仓库 docs/README.md
├── .agents/                  # 推荐：项目 skill
│   ├── README.md             # 模板：本仓库 .agents/README.md
│   └── skills/<name>/SKILL.md
└── openspec/                 # 可选：仅场景三（OpenSpec）
    └── changes/<change-name>/
```

| 路径 | 放什么 | 是否必做 |
| --- | --- | --- |
| `AGENTS.md` | AI 协作六条底线 | 必做 |
| `docs/` | 架构、API、业务等长期规范 | 推荐 |
| `.agents/skills/` | 项目 skill；索引用 `.agents/README.md` | 有 skill 时 |
| `openspec/changes/` | 进行中需求的 proposal / specs / tasks | 仅 OpenSpec |

**边界**：`docs/` ≠ 需求变更。长期约定进 `docs/`；某次需求的澄清、验收、任务进 OpenSpec change 或 issue/PR。详见 [`docs/README.md`](./docs/README.md)。

### 初始化清单

1. 复制 `AGENTS.md` 到项目根目录。
2. 创建 `docs/`，复制本仓库 [`docs/README.md`](./docs/README.md)。
3. 创建 `.agents/`，复制本仓库 [`.agents/README.md`](./.agents/README.md)；有 skill 时建 `.agents/skills/<name>/SKILL.md` 并更新索引。
4. （场景三）在业务项目安装 OpenSpec，需求文档进 `openspec/changes/`。

## 场景一：只用 AGENTS.md（最小落地）

**适合**：个人项目、小团队、或只想约束 AI 协作底线。

1. 完成上文「初始化清单」第 1～3 步（场景一至少完成第 1 步；有规范或 skill 时完成 2～3）。
2. 若项目已有 `AGENTS.md`，先 diff 差异，确认合并方案后再写入。

**典型用法示例**

| 改动类型 | 你要做的事 | AI 应遵守的底线 |
| --- | --- | --- |
| 修错别字 / 局部文案 | 直接描述改动 | 最小验证即可 |
| 修不改变对外行为的 bug | 说明复现与预期 | 验证相关测试或构建 |
| 改 API / 权限 / 流程 | 先在 issue 或 PR 描述写清范围与验收，再改代码 | 需求可追溯 + 完成前验证 |

**与各类 AI 工具配合**：`AGENTS.md` 放项目根目录作通用底线；项目规范放 `docs/`；项目 skill 放 `.agents/skills/`，目录索引维护在 `.agents/README.md`。不要把规范或 skill 拆到各工具自有配置里，也不要另写一套与 `AGENTS.md` 冲突的口径。

## 场景二：AGENTS.md + Issue / PR（轻量需求治理）

**适合**：有基本评审流程、但暂不引入 OpenSpec 的团队。

1. 复制 `AGENTS.md`（同场景一）。
2. 会改变外部可观察行为时，先在 issue 或 PR 描述中写清：
   - 变更范围
   - 验收标准
   - 验证方式
   - 未验证项与剩余风险
3. 需求变更时：更新正文为当前口径，**追加**变更记录，不覆盖历史。

OpenSpec 的 change 目录可等价替换为 issue / PR / 任务卡，流程细节见 [`新需求开发.md`](./新需求开发.md) 与 [`需求变更.md`](./需求变更.md) 文末「未启用 OpenSpec 时」。

---

## 场景三：全套工具链（OpenSpec + CodeGraph + Superpowers）

**适合**：需要规格化需求、影响分析与执行纪律的中大型团队。

1. 复制 `AGENTS.md` 到业务项目根目录；按需安装两个项目流程 skill（**安装 skill 不依赖**三项工具），见 [`README.md` 项目流程 skill](./README.md#项目流程-skill可选)。
2. 在**业务项目**安装 **OpenSpec → CodeGraph → Superpowers**（**使用 skill 跑完整流程**时需要；安装提示词见 [`README.md`](./README.md#通过-ai-安装)）。
3. 按流程操作（手动 `@` 调用 skill，不自动触发）：
   - 新需求 → [`新需求开发.md`](./新需求开发.md)
   - 需求变更 → [`需求变更.md`](./需求变更.md)
   - 三种命令写法 → [`流程公共约定.md`](./流程公共约定.md)

分工记忆口诀：**AGENTS 定底线 · OpenSpec 管需求 · CodeGraph 管影响 · Superpowers 管方法**。

---

## 选型建议

```text
只想管 AI 别乱提交、别伪造成功     → 场景一
有评审但不想上工具链               → 场景二
要规格、验收、变更可追溯闭环         → 场景三
```

无论选哪种场景，[`AGENTS.md`](./AGENTS.md) 六条底线均适用。
