# 快速开始

> 本页只用于选择接入场景与查找后续入口。通用工程规则见 [`AGENTS.md`](./AGENTS.md)，安装、初始化与接入方式见 [`README.md`](./README.md)。

## 场景一：只需要稳定的工程底线

**适合**：个人项目、小团队，以及已有成熟流程、只需约束 AI 协作底线的项目。

- 最小安装：[`README.md` §1](./README.md#1-复制到业务项目)；AI 接入与升级提示词：[`README.md` §3](./README.md#3-用-ai-接入或升级-vibe-coding-rule)
- 通用工程规则：[`AGENTS.md`](./AGENTS.md)
- 项目规范与项目级 skill 的按需接入：[`README.md` §1](./README.md#1-复制到业务项目)

## 场景二：需要轻量需求治理，但不使用 OpenSpec

**适合**：使用 Issue、PR、任务卡等既有载体，希望保留需求与验证记录的团队。

- 最小安装：[`README.md` §1](./README.md#1-复制到业务项目)；AI 接入与升级提示词：[`README.md` §3](./README.md#3-用-ai-接入或升级-vibe-coding-rule)
- 新需求导航：[`新需求开发.md` 的“未启用 OpenSpec”路径](./新需求开发.md#未启用-openspec-时)
- 需求变更导航：[`需求变更.md` 的“未启用 OpenSpec”路径](./需求变更.md#未启用-openspec-时)
- 通用留痕与验证规则：[`AGENTS.md` §3](./AGENTS.md#3-需求可追溯)

## 场景三：需要 OpenSpec / CodeGraph / Superpowers 协作

**适合**：复杂需求、跨模块改动、多人协作，以及需要更强需求治理或代码分析的项目。

- 完整安装：[`README.md` §1](./README.md#1-复制到业务项目)；可选工具安装与初始化：[`README.md` §4](./README.md#4-可选工具链通过-ai-安装)
- 工具启用条件、调用方式、限制与 fallback：[`docs/规范/可选工具协作.md`](./docs/规范/可选工具协作.md)
- 新需求导航：[`新需求开发.md`](./新需求开发.md)
- 需求变更导航：[`需求变更.md`](./需求变更.md)

## 选型建议

| 你的情况 | 推荐场景 |
| --- | --- |
| 只需要通用工程约束 | 场景一 |
| 需要需求记录，但不使用 OpenSpec | 场景二 |
| 需要可选工具协作 | 场景三 |

拿不准时先选择场景一；需要更多治理或工具能力时，再沿对应入口扩展。
