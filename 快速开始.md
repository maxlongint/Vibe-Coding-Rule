# 快速开始

> Vibe Coding Rule 提供一套完整协作规范，工具按需启用。通用工程规则见 [`AGENTS.md`](./AGENTS.md)，接入与升级方式见 [`README.md`](./README.md)。

## 1. 工具启用模型

OpenSpec、CodeGraph、Superpowers 都是可选工具。开始任务时先确定唯一需求载体；工具启用、职责、组合方式、可用性检查和失败处理见 [`docs/规范/工具协作.md`](./docs/规范/工具协作.md)。

## 2. 接入完整规范

按 [`README.md` 的“接入业务项目”](./README.md#接入业务项目)复制整套规范。项目只提供这一种接入方式。

## 3. 选择工作入口

| 任务 | 入口 |
| --- | --- |
| 新功能或大改动 | [`新需求开发.md`](./新需求开发.md) |
| 修改已有需求 | [`需求变更.md`](./需求变更.md) |
| 查询工具职责、协作顺序和限制 | [`docs/规范/工具协作.md`](./docs/规范/工具协作.md) |
| 使用 UI 设计输入重组两阶段任务 | [`docs/规范/UI任务重组.md`](./docs/规范/UI任务重组.md) |
| 查询长期项目规范 | [`docs/README.md`](./docs/README.md) |

无论从哪个入口开始，都先确定唯一需求载体。OpenSpec、CodeGraph 与 Superpowers 仅在已启用时按需组合使用；最终口径、设计、计划和验证结论都回填同一载体。
