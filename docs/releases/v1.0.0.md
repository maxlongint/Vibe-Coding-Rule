## Vibe Coding Rule v1.0.0

**Vibe Coding Rule** 是一套面向 AI 辅助开发的跨项目协作规则包：前后端通用、工具无关。复制 [`AGENTS.md`](https://github.com/maxlongint/Vibe-Coding-Rule/blob/main/AGENTS.md) 即可在 Minimal 档位独立使用，无需 OpenSpec、CodeGraph、Superpowers 或任何特定 AI 插件。

### Highlights

- **`AGENTS.md` 自洽底线**：需求治理、工程守则、验证、提交权限、规则优先级 —— 单文件即可落地。
- **三档复制模型**：Minimal / Standard / Full，团队可按成熟度选用。
- **可选工具链**：OpenSpec、CodeGraph、Superpowers 均可缺省，缺失时按文档 fallback。
- **项目级技能包**：复杂需求与需求变化同步（Full 档位）。
- **流程教程与示例**：OpenSpec 新需求/调整步骤、Superpowers 配合指南、SSO 教学示例。

### 规则包内容

| 路径 | 说明 |
| --- | --- |
| `AGENTS.md` | 强制协作章程（所有档位） |
| `docs/规范/` | 需求治理、AI 工具链、轻量记录模板 |
| `docs/安装与AI提示词.md` | 复制步骤与各档位 AI 安装提示词 |
| `.agents/skills/` | `executing-complex-requirements`、`updating-requirement-changes` |
| `.github/pull_request_template.md` | PR 模板（Standard 起推荐） |
| `OpenSpec*.md`、`Superpowers*.md` | 流程教程 |
| `SSO 单点登录复杂需求执行步骤示例.md` | 复杂需求教学（虚构场景） |

### 快速开始（Minimal）

```bash
git clone https://github.com/maxlongint/Vibe-Coding-Rule
cp Vibe-Coding-Rule/AGENTS.md /path/to/your-project/
```

将 `AGENTS.md` 放在业务项目根目录。Cursor 等工具会默认识别。

Windows（PowerShell）：

```powershell
git clone https://github.com/maxlongint/Vibe-Coding-Rule
Copy-Item Vibe-Coding-Rule\AGENTS.md -Destination C:\path\to\your-project\
```

### 档位说明

| 档位 | 复制 |
| --- | --- |
| **Minimal** | `AGENTS.md` |
| **Standard** | Minimal + `docs/规范/需求治理与文档结构.md` + PR 模板（推荐） |
| **Full** | Standard + 全部 `docs/规范/` + `.agents/skills/` |

详见 [README](https://github.com/maxlongint/Vibe-Coding-Rule#三档选择) 与 [安装与 AI 提示词](https://github.com/maxlongint/Vibe-Coding-Rule/blob/main/docs/安装与AI提示词.md)。

### 已知限制
