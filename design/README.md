# 项目设计资料（design/）

本目录存放 UI、页面、交互和视觉设计资料。项目规范放在 `docs/`；程序运行时使用的图片、图标、字体等资源放在项目既有资源目录，不放在本目录。

## 目录结构

```text
design/
├── README.md       # 本文件：设计资料说明与检索约定
├── specs/          # 设计方案、页面说明、交互说明
├── ui/             # UI 稿、页面截图、标注图
└── assets/         # 仅供设计和文档引用的图片、图标等资源
```

子目录按实际需要创建，不为空目录添加占位文件；资料增多后可再按页面、组件或流程细分。普通设计文件通过目录和文件名查找，无需维护逐文件索引。
